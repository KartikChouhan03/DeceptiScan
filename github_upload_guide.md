# Guide: Uploading DeceptiScan to GitHub

This guide walks you through the process of uploading your project to GitHub for the first time, ensuring that large files (like models and virtual environments) and sensitive directories (like node_modules and env files) are correctly excluded.

---

## 1. Prerequisites
Before uploading to GitHub, ensure you have:
1. A **GitHub Account** (https://github.com).
2. **Git installed** on your computer. If not, download it from [git-scm.com](https://git-scm.com/).
3. A new **Repository** created on GitHub:
   - Go to GitHub and click **New** repository.
   - Name it `DeceptiScan` (or your choice).
   - **Do NOT** check "Add a README", "Add .gitignore", or "Choose a license" since we already have these files in our project.
   - Keep the repository **Public** or **Private** as you prefer.
   - Click **Create repository**.
   - Copy the repository's remote URL (it will look like `https://github.com/your-username/DeceptiScan.git`).

---

## 2. Git Ignored Files (.gitignore)
We have already updated and optimized the `.gitignore` files in your project directory:
* [Root .gitignore](file:///d:/Deceptiscan/DECEPTISCAN---BROWSER-EXTENSION-main/Deceptiscan/.gitignore)
* [Extension-specific .gitignore](file:///d:/Deceptiscan/DECEPTISCAN---BROWSER-EXTENSION-main/Deceptiscan/extension/.gitignore)

### What is ignored and why?
* **`node_modules/`**: Contains thousands of downloaded NPM library files. It is generated automatically via `npm install` and should never be uploaded.
* **`.venv/`, `venv/`, `env/`**: Your Python virtual environment containing the Python runtime and library dependencies. It is huge and should be recreated on other machines using `pip install -r requirements.txt`.
* **`backend/models/` and `*.safetensors` / `*.bin`**: The trained ML models (such as the ~738MB `model.safetensors` for DeBERTa-v3) exceed GitHub's single-file upload limit of 100MB. Git will refuse to push them unless you use Git LFS (Large File Storage). Instead of uploading models to git, they should be stored on an external release hosting service (like Hugging Face or Google Drive) or kept locally, while checking in only the loading scripts.
* **`.env` files**: Contain local environment secrets, API keys, or custom configuration endpoints.
* **`.plasmo/` & `build/`**: Build artifacts generated by the compiler.
* **`.vscode/`, `.DS_Store`**: Personal editor configurations and OS-specific files.

## 3. Step-by-Step Command Line Guide

Open a terminal (like PowerShell, Git Bash, or CMD) in your project root folder:
`d:\Deceptiscan\DECEPTISCAN---BROWSER-EXTENSION-main\Deceptiscan`

### Step 3.1: Initialize Git Repository
Initialize the directory as a Git repository.
```bash
git init
```

### Step 3.2: Verify Git Status
Check the status to ensure no massive model files or `.venv` folders are being tracked.
```bash
git status
```
*You should only see `.github/`, `.gitignore`, `LICENCE`, `README.MD`, and `extension/` listed as untracked files.*

### Step 3.3: Stage the Files
Stage all the files to be committed:
```bash
git add .
```

### Step 3.4: Commit the Files
Commit the files with an initial message:
```bash
git commit -m "initial commit: add browser extension, backend server, and optimized UI"
```

### Step 3.5: Set the Main Branch
Set your default branch name to `main`:
```bash
git branch -M main
```

### Step 3.6: Link your GitHub Repository
Connect your local repository to your remote GitHub repository. (Replace the URL with your copied GitHub URL):
```bash
git remote add origin https://github.com/your-username/DeceptiScan.git
```

### Step 3.7: Push to GitHub
Push your local code to the GitHub repository:
```bash
git push -u origin main
```

## 4. How to share the large model with others
Since the DeBERTa model (`model.safetensors`) is ignored:
1. **Option A (Recommended)**: Upload the folder `deberta-v3-detector` to **Hugging Face Hub** (which is free and supports large model weights natively). Then, users can download the model directly.
2. **Option B (Google Drive/OneDrive)**: Zip the `deberta-v3-detector` folder, upload it to cloud storage, and share the link in the `README.md` so users can download and paste it into `extension/backend/models/` manually.
